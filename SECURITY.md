# 보안 정책 (Security Policy)

## 지원 버전

현재 보안 업데이트를 지원하는 버전:

| 버전 | 지원 여부 |
|------|-----------|
| main (최신) | 지원 |
| develop | 지원 |
| 이전 릴리스 | 미지원 |

## 취약점 신고 방법

보안 취약점을 발견하셨다면, **공개 이슈로 등록하지 마시고** 다음 방법으로 신고해 주세요:

1. **GitHub Security Advisories**: [보안 권고 생성](https://github.com/GovOn-Org/GovOn/security/advisories/new)을 통해 비공개로 신고
2. **이메일**: 프로젝트 관리자에게 직접 연락

### 신고 시 포함할 정보

- 취약점의 유형 및 위치 (파일 경로, 함수명 등)
- 취약점을 재현할 수 있는 단계별 설명
- 취약점의 예상 영향 범위
- (가능한 경우) 수정 방안 제안

## 보안 관련 주의사항

### 민감 정보 관리

이 프로젝트에서 특히 주의해야 할 민감 정보:

- **API 키/토큰**: HuggingFace, WandB 등의 API 키를 코드에 포함하지 마세요
- **모델 가중치 경로**: 로컬 파일 시스템 경로를 하드코딩하지 마세요
- **데이터셋 경로**: 개인 정보가 포함된 민원 데이터 경로를 노출하지 마세요
- **서버 정보**: 추론 서버의 IP 주소, 포트 등을 공개하지 마세요

### .gitignore 필수 항목

다음 파일/디렉토리가 `.gitignore`에 포함되어 있는지 확인하세요:

```
# 환경 변수 및 인증 정보
.env
*.key
*credentials*

# 모델 가중치 (대용량)
*.bin
*.safetensors
*.gguf

# 데이터셋 (민감 정보 포함 가능)
data/raw/
data/private/
```

## 응답 절차

1. **확인**: 신고 접수 후 3일 이내에 확인 응답
2. **분석**: 취약점 심각도 평가 및 영향 범위 파악
3. **수정**: 패치 개발 및 테스트
4. **공개**: 수정 완료 후 보안 권고(Security Advisory) 발행

## 감사의 말

보안 취약점을 책임감 있게 신고해 주시는 모든 분들께 감사드립니다.
