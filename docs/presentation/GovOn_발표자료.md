## 1. GovOn이 뭐예요?

### 🤔 문제 상황부터 볼게요

여러분이 **생일 파티**를 준비한다고 생각해봐요.

- 🍰 **케이크**는 빵집에 가야 하고
- 🎈 **풍선**은 문구점에 가야 하고
- 🎁 **선물**은 장난감 가게에 가야 하고
- 🥤 **음료수**는 편의점에 가야 해요

가게가 **다 따로따로** 있어서 **일일이 다 돌아다녀야** 해요. 피곤하죠?

### 💡 GovOn의 아이디어

**"가게를 한꺼번에 부탁할 수 있는 심부름꾼이 있다면 어떨까?"**

> 👦 "심부름꾼아, 생일 파티 준비해줘!"
>
> 🤖 심부름꾼: "알겠어요! 제가 빵집, 문구점, 장난감 가게, 편의점 다 다녀올게요."

**GovOn은 바로 이런 똑똑한 심부름꾼이에요.** 단, 생일 파티가 아니라 **정부 일**을 도와줘요.

### 🏛️ 현실 세계 버전

대한민국에는 **수많은 정부 기관**이 있어요. 각 기관마다 **따로따로 인터넷 창구**가 있죠.

- 민원 → A 기관 창구
- 법률 정보 → B 기관 창구
- 통계 자료 → C 기관 창구
- 인구 정보 → D 기관 창구

**GovOn**은 이 모든 창구를 **한 번에** 대신 다녀와서, 여러분이 원하는 답을 **하나의 대답**으로 만들어줘요.

### 🎯 한 줄 요약

> **GovOn = 흩어진 정부 시스템을 하나의 채팅창에서 물어볼 수 있게 해주는 AI 심부름꾼**

## 2. 왜 만들었어요?

### 😩 공무원 아저씨/아주머니의 하루

매일매일 이런 일이 반복돼요.

1. 시민이 "우리 동네 도로 파손 신고했는데 어떻게 처리돼요?" 하고 물어봐요.
2. 공무원은 **3개 시스템을 열고** → **5번 검색하고** → **2개 법령을 읽고** → **답변을 작성**해요.
3. 하루에 이런 질문이 **수백 개**씩 들어와요.

### ⏰ 걸리는 시간

- 검색: 10분
- 법령 확인: 15분
- 답변 작성: 20분
- **총 45분** × 수백 건 = **야근 확정** 😭

### ✨ GovOn이 있으면?

> 👩 공무원: "GovOn, 도로 파손 민원 처리 절차 알려줘."
>
> 🤖 GovOn: (5초 후) "여기 절차랑 법적 근거입니다!"

**45분 → 5초**. 공무원은 **사람만 할 수 있는 일**에 집중할 수 있어요.

### 🎁 누구에게 좋은가요?

| 대상 | 좋은 점 |
|---|---|
| 👩‍💼 **공무원** | 단순 조회 업무 시간을 확 줄여요 |
| 👨‍👩‍👧 **시민** | 복잡한 절차를 쉽게 안내받아요 |
| 🏢 **정부 기관** | 서비스 품질이 올라가요 |
| 🌏 **국가** | 디지털 강국 이미지 업! |

## 3. 전체 모습 보기 — 우리 팀이 만든 큰 그림

![GovOn 전체 시스템 아키텍처](./images/govon_전체_시스템_아키텍처.svg)

### 🏢 건물 비유로 이해하기

GovOn은 **4층짜리 건물**이에요.

#### 🏠 4층: 손님 맞이하는 곳 (클라이언트)

손님(사용자)이 **직접 만나는 공간**이에요.

- **GovOn CLI** 🖥️ — 검은 화면(터미널)에서 쓰는 채팅창
- **웹 브라우저** 🌐 — 크롬 같은 데서 쓰는 채팅창

#### 🚪 3층: 안내 데스크 (API 게이트웨이)

손님이 온 걸 **접수하고 분류**하는 곳이에요.

- 누구인지 확인(비밀번호 체크 🔑)
- 너무 자주 오면 조금 기다리게 하기(30번/분 제한 ⏱️)
- 어느 방으로 안내할지 결정

#### 🧠 2층: GovOn의 뇌 (에이전트 코어)

**가장 중요한 곳**이에요. 여기서 **진짜 생각**이 일어나요.

- 기억 창고 💾 (SQLite) — 이전 대화 저장
- 생각 루프 🔄 (LangGraph) — 생각하고 → 도구 쓰고 → 또 생각하고
- 도구함 🧰 — 7개 도구 준비
- 설정 노트 📒 — 규칙 적어둠

#### ⚡ 1층: 힘 쓰는 곳 (추론/데이터)

실제로 **말을 만들어내고**, **정보를 가져오는** 곳이에요.

- **EXAONE 4.0 모델** 🤖 — 한국어 잘하는 거대 AI
- **LoRA 어댑터** 🎓 — 분야별 전문가 모자 (법률 모자, 행정 모자)
- **정부 API** 📡 — 공공데이터포털에서 정보 가져오기

### 🔍 옆에는 뭐가 있어요?

건물 옆에는 **관리실**이 있어요.

- 🐳 **도커** — 건물을 어디든 옮길 수 있는 컨테이너
- 📊 **Grafana** — 건물 상태를 실시간으로 보는 대시보드
- 🔧 **GitHub Actions** — 건물 자동 점검/수리 로봇

### 🎯 한 줄 요약

> **사용자 질문 → 접수 → 생각 → 힘써서 대답 만들기**
> 이 4단계가 4개 층에 딱 맞춰 만들어져 있어요.

## 4. GovOn의 뇌는 어떻게 생각해요?

![LangGraph ReAct v3 토폴로지](./images/govon_langgraph_react_v3_토폴로지.svg)

### 🧸 요리사 비유

GovOn의 뇌는 **요리사 로봇**처럼 일해요.

#### 👨‍🍳 요리 단계

1. **🔓 재료 꺼내기 (session_load)**
   - 손님이 전에 주문한 메뉴 기억 꺼내기
   - 너무 많으면 요약해서 머리에 저장
   - "아, 이 손님은 매운 걸 좋아했지!"

2. **🤔 메뉴 정하기 (agent)**
   - 손님 요청을 보고 "이 요리엔 냄비가 필요하네, 채소도 필요하고..."
   - 필요한 도구를 정해요

3. **🔨 도구 쓰기 (tools)**
   - 냄비 꺼내고 🍲
   - 채소 썰고 🥬
   - 물 끓이고 💧

4. **🔁 다시 생각 (agent로 돌아감)**
   - "이제 간을 더 할까? 아니면 바로 낼까?"
   - 필요하면 다시 도구 꺼내기!

5. **🍽️ 완성 (synthesize)**
   - 최종 음식 플레이팅 ✨
   - "여기 있습니다, 손님!"
   - 레시피도 기록해두기 📝

### 🔄 **가장 중요한 건 "반복"이에요!**

```
생각 → 도구 → 생각 → 도구 → 생각 → 완성
```

**최대 10번까지** 반복할 수 있어요. 왜? 복잡한 요리는 한 번에 안 되니까!

### 📋 머릿속에 뭐가 들어있어요? (State)

요리사가 항상 들고 다니는 **체크리스트**가 있어요.

- 📞 **session_id**: 손님 번호
- 💬 **messages**: 지금까지 주고받은 대화
- 🛠️ **pending_tool_calls**: 쓸 예정인 도구들
- 🔢 **iteration_count**: 몇 번 반복했는지
- ✋ **approval_status**: 손님 허락받았는지

### 🧠 똑똑한 기억 관리

머리가 꽉 차면 어떻게 해요? **요약**해요!

- 📏 기억 용량: 4500 토큰 (글자 약 2250자)
- 📦 60%가 차면 → 오래된 건 **요약**
- ⭐ 최근 6개 대화는 **항상 그대로** 간직

### 🎯 한 줄 요약

> **생각 → 도구 → 생각 → 도구 → 완성**의 반복!
> 최대 10번 반복하면서 복잡한 질문도 풀어내요.

## 5. GovOn이 가진 도구상자

![도구 생태계 카탈로그](./images/govon_도구_생태계_카탈로그.svg)

### 🧰 7개의 마법 도구

GovOn은 **7개의 도구**를 가지고 있어요. **3개의 상자**로 정리돼 있어요.

### 📦 1번 상자: **검색 도구** (정보 찾기)

#### 🔍 api_lookup
> "공공데이터포털에 가서 민원 정보 찾아와!"
>
> 시민 불만 신고 같은 걸 검색해요.

### 📦 2번 상자: **분석 도구** (계산/정리)

#### 📈 issue_detector
> "요즘 뜨거운 이슈가 뭐야?"
>
> 트렌드를 찾아줘요.

#### 📊 stats_lookup
> "이 동네 인구 통계 줘."
>
> 숫자 자료를 찾아줘요.

#### 🔤 keyword_analyzer
> "이 글에서 중요한 단어가 뭐야?"
>
> 키워드를 뽑아줘요.

#### 👥 demographics_lookup
> "이 지역 나이대별 인구가 어떻게 돼?"
>
> 인구 정보를 줘요.

### 📦 3번 상자: **전문가 도구** (글쓰기) 🎓

여기는 **특별한 도구**가 있어요. **전문가 모자(LoRA)**를 쓴 도구들이에요!

#### 👩‍⚖️ legal_adapter (법률 전문가)
> "법률 답변을 써줘."
>
> 법적 근거까지 달아서 초안을 만들어줘요.
> ⚠️ **사람 허락 필요!**

#### 🏛️ public_admin_adapter (행정 전문가)
> "행정 절차 답변을 써줘."
>
> 민원/허가 관련 답변을 만들어줘요.
> ⚠️ **사람 허락 필요!**

### 🛡️ 왜 "허락"이 필요해요?

**중요한 답변**은 AI가 혼자 내보내면 안 돼요. 법률이나 행정은 **잘못 말하면 큰일**나니까요!

> 👨‍💼 "AI가 만든 초안을 **공무원이 확인**한 다음에 손님에게 보내자!"

이게 바로 **사람이 중간에 관여하는(Human-in-the-Loop)** 안전 장치예요.

### 🔄 도구가 자동으로 늘어나요!

신기한 점: **설정 파일(YAML)**만 바꾸면 도구가 **새로 생겨요!**

```yaml
# 이 파일을 수정하면 도구가 자동 등록돼요
adapters:
  새로운_전문가:
    path: "허깅페이스/새_어댑터"
    domain: "새 분야"
```

**코드 수정 없이** 도구를 추가할 수 있어요! 🎁

### 🎯 한 줄 요약

> **검색 1개 + 분석 4개 + 전문가 2개 = 총 7개 도구!**
> 전문가 도구는 사람 허락을 받고 써요.

## 6. 실시간으로 대답이 만들어지는 모습

![SSE 이벤트 타임라인](./images/govon_SSE_이벤트_타임라인.svg)

### 🎬 영화관 스크린 비유

GovOn은 **답을 한 번에** 보내지 않아요. **영화 자막**처럼 **실시간으로 조금씩** 보내줘요.

### 🍿 왜 그렇게 해요?

만약 여러분이 피자를 시켰는데:

- 🅰️ **옵션 A**: 30분 동안 아무 소식 없다가 갑자기 도착 😴
- 🅱️ **옵션 B**: "지금 만드는 중...", "오븐에 들어갔어요", "배달 출발!" 실시간 알림 😊

당연히 **B가 훨씬 좋죠?** GovOn도 그렇게 해요!

### 📺 화면에 뜨는 4가지 종류의 이벤트

시간축을 따라 **4개 레인**으로 이벤트가 흘러요.

#### 🔵 레인 1: 생각 중 (thinking)
> "GovOn이 머리 굴리는 중이에요..."

- `thinking_start` — 생각 시작 🧠
- `thinking_delta` — 생각 내용 조금씩 (빠르게 여러 개)
- `thinking_end` — 어떤 도구 쓸지 결정 완료

#### 🟠 레인 2: 도구 사용 중 (tool)
> "GovOn이 도구를 꺼냈어요!"

- `tool_start` — "api_lookup 도구 사용 시작" 🔨
- `tool_end` — "완료! 정보 가져왔어요" ✅

#### 🟢 레인 3: 답변 중 (response)
> "이제 답을 조금씩 써요..."

- `response_delta` — 답변을 한 글자씩 스트리밍
- (마치 타자 치듯이 뚜드득뚜드득)

#### 🟣 레인 4: 종료 (complete)
> "끝!"

- `run_complete` — 최종 답변 + 증거 자료 + 메타데이터

### ⏱️ 실제 타임라인 예시 (3.5초 만에 끝!)

```
0.0초 ──▶ "생각 시작..." 🔵
0.5초 ──▶ "도구 2개 쓸 거예요" 🔵
0.55초 ─▶ "api_lookup 시작" 🟠
1.15초 ─▶ "api_lookup 완료" 🟠
1.20초 ─▶ "public_admin_adapter 시작" 🟠
2.05초 ─▶ "public_admin_adapter 완료" 🟠
2.10초 ─▶ "또 생각 시작 (2번째 반복)" 🔵
2.80초 ─▶ "답변 작성 중..." 🟢
3.50초 ─▶ "완성! 🎉" 🟣
```

### 🎯 한 줄 요약

> **실시간 스트리밍**으로 **3.5초 만에** 답변 완성!
> 사용자는 "지금 뭐 하는 중인지" **눈으로 바로** 볼 수 있어요.

## 7. "중요한 일은 사람이 허락해야 해요" — 안전 장치

![HITL 승인 플로우](./images/govon_HITL_승인_플로우.svg)

### 🚦 횡단보도 신호등 비유

운전하다가 **빨간불**을 보면 멈추죠? AI도 중요한 순간엔 **멈춰야** 해요.

### ⚠️ 언제 멈춰요?

**법률**이나 **행정** 같은 중요 도구를 쓸 때 **자동으로 멈춰요.**

예시 시나리오:
> 👩 공무원: "민법 제750조 해석해줘."
>
> 🤖 GovOn: (legal_adapter 꺼내려 함) → ✋ **정지!**
>
> 💬 "이 도구는 법률 답변이에요. 쓸까요? [승인] [거부] [취소]"

### 🔄 어떻게 동작해요?

#### 1️⃣ **발견 단계**
> 🤖 "어? 이 도구는 허락이 필요한 거네!"

#### 2️⃣ **일시정지 (interrupt)**
> ⏸️ GovOn이 일을 멈추고, 현재 상태를 저장해요.
> 마치 비디오 게임에서 **세이브하고 잠깐 쉬는** 것처럼요!

#### 3️⃣ **사용자에게 질문**
사용자 화면에 이런 창이 떠요:
```
┌─ 승인 요청 ─────────────────┐
│ 종류: 법률 답변 초안 작성    │
│ 도구: legal_adapter         │
│                             │
│ [✅ 승인] [❌ 거부] [🚫 취소] │
└─────────────────────────────┘
```

#### 4️⃣ **3가지 선택지**

| 선택 | 다음 동작 |
|---|---|
| ✅ **승인** | 도구 실행 → 답변 만들기 |
| ❌ **거부** | "안 쓸게요, 다른 방법 생각해볼게요" → 다시 생각 |
| 🚫 **취소** | "알겠어요, 여기서 끝낼게요" → 종료 |

#### 5️⃣ **재개**
선택에 따라 GovOn이 **저장한 지점부터 다시 시작**해요.

### 🎯 왜 이게 중요해요?

- ⚖️ 법률 답변은 **잘못되면 큰 문제**
- 📋 행정 답변은 **실제 시민에게 영향**
- 🤝 **AI + 사람** = 가장 안전!

### 🎯 한 줄 요약

> **중요한 일에는 "사람이 확인"하는 과정**이 들어있어요.
> AI만 믿지 않고, 사람과 **함께 일하는** 시스템이에요!

## 8. GovOn이 똑똑해지는 공부법 — LoRA 학습

![LoRA 학습 파이프라인](./images/govon_LoRA_학습_파이프라인.svg)

### 🎓 전문가 모자 만들기 비유

GovOn의 기본 뇌(**EXAONE 4.0**)는 **일반 상식**을 많이 알아요. 하지만 **법률**이나 **행정**은 더 깊게 공부해야 해요.

**"일반 지식 + 전문 지식"** → 이게 바로 **LoRA 어댑터**!

마치 요리사가 **"평소엔 모자 없이, 한식 만들 땐 한식 모자, 양식 만들 땐 양식 모자"**를 바꿔 쓰는 것처럼요! 🍳

### 📚 6단계 학습 과정

#### 1️⃣ 공부 자료 모으기 📖
- AI Hub에서 자료 다운로드
- **민법** 문답 (71841)
- **지식재산권** 문답 (71843)
- **형법** 문답 (71848)
- 합치면 **27만 개**의 질문-답변 쌍!

#### 2️⃣ 자료 정리하기 ✂️
- 문장 조각내기 (tokenization)
- 이상한 데이터 골라내기
- 허깅페이스 데이터셋으로 업로드

#### 3️⃣ 공부방 준비하기 🏫
- **A100 GPU 80GB** (엄청 큰 학습 컴퓨터)
- 프레임워크: **Unsloth + TRL**
- 4-bit로 **모델 압축** (메모리 절약!)
- LoRA: rank=16, alpha=32

#### 4️⃣ 실제 공부 📝
- 365단계 학습
- 약 **7시간** 소요 (A100 기준)
- 시험 점수(Loss): **2.334 → 0.889** (많이 향상!)

> 점수가 낮을수록 잘하는 거예요. 아이가 받아쓰기 10개 중에서 틀린 개수가 줄어드는 것과 같아요!

#### 5️⃣ 증명서 받기 🏆
- 학습 끝나면 **작은 가중치 파일** 생성
- 허깅페이스에 업로드: `siwo/govon-legal-adapter`
- 설정 파일에 등록

#### 6️⃣ 실전 투입! 🚀
- GovOn 서버 시작할 때 자동으로 로드
- 사용자가 법률 질문하면 → **법률 모자** 착용!
- 답변 완료하면 모자 벗기

### 🎯 왜 전체 모델을 다시 안 만들어요?

**전체 모델 새로 학습하기** (❌ 비효율):
- 수백 GB의 파라미터 전부 수정
- 몇 달 걸림

**LoRA 방식** (✅ 효율적):
- **작은 어댑터만** 수정
- 몇 시간 만에 끝
- 적은 비용
- **여러 어댑터를 바꿔끼기** 가능!

## 10. 모든 규칙이 적힌 노트 — 설정 파일

![구성 단일 진리원](./images/govon_구성_단일_진리원.svg)

### 📒 학급 규칙 노트 비유

우리 반에는 **하나의 규칙 노트**가 있어요. 모든 학생이 그 노트를 봐요.

- 📝 담임 선생님이 규칙을 바꿔요
- 👨‍🏫 체육 선생님도 그 노트를 봐요
- 👩‍🎨 미술 선생님도 그 노트를 봐요
- 👨‍🍳 급식 아줌마도 그 노트를 봐요

**"규칙 = 한 곳에만 적혀 있음"** → 안 헷갈려요!

### 📘 GovOn의 2개 규칙 노트

#### 1️⃣ `config/govon.yaml` — **전체 규칙 노트**

```yaml
# 생성 관련
generation:
  max_tokens: 512          # 답변 최대 길이
  temperature: 0.7         # 창의성 정도

# 서빙 관련
serving:
  gpu_memory_utilization: 0.90   # GPU 90%까지 써!
  max_model_len: 8192            # 최대 8192 토큰까지 처리

# 문맥 관리
context:
  max_iterations: 10             # 최대 10번 반복
  agent_input_budget: 4500       # 메시지 예산

# 도구 관리
tools:
  defaults:
    timeout_sec: 10.0            # 도구는 10초 안에 끝내기
```

#### 2️⃣ `config/adapters.yaml` — **전문가 모자 노트**

```yaml
adapters:
  public_admin:                  # 행정 모자
    path: "umyunsang/govon-civil-adapter"
    requires_approval: true      # 사람 허락 필요!

  legal:                         # 법률 모자
    path: "siwo/govon-legal-adapter"
    requires_approval: true      # 사람 허락 필요!
```

### 🔄 이 노트를 보는 사람(코드)들

6명의 **소비자**가 이 노트를 참고해요!

1. 🎚️ **runtime_config.py** — 실행 환경 설정
2. 📚 **adapter_registry.py** — 어댑터 목록 로드
3. 🧠 **graph/nodes.py** — 문맥 관리 규칙 적용
4. 🧰 **graph/tools/__init__.py** — 도구 타임아웃 설정
5. ⚡ **vLLM entrypoint.sh** — GPU/모델 설정
6. 🚪 **api_server.py** — API 제한 설정

### 🌡️ 환경변수로 즉석 변경

노트를 안 바꿔도 **터미널에서 즉석 변경**도 가능해요!

```bash
# "최대 반복 횟수를 15로 바꿔!"
export GOVON_CONTEXT_MAX_ITERATIONS=15

# "답변 길이를 1024로!"
export GOVON_GENERATION_MAX_TOKENS=1024
```

### 🎯 왜 한 곳에 모아요?

| ❌ 안 좋은 방식 | ✅ 좋은 방식 (GovOn) |
|---|---|
| 여기저기 설정이 흩어져 있음 | **한 곳**에 모여 있음 |
| 뭘 바꿔야 할지 모름 | **YAML만 바꾸면** 전부 반영 |
| 실수로 불일치 발생 | **항상 일관성 유지** |

### 🎯 한 줄 요약

> **2개의 YAML 파일 = 모든 규칙의 정답지**
> 바꾸고 싶으면 이 파일만 고치면 돼요!

#### 🤖 6층: 에이전트 런타임 (Agent Runtime)

- 승인 UX

#### 🎯 8층 (꼭대기): 가치 (Value)
> "우리가 만드는 **궁극적인 가치**"

> **"한국 정부 DX의 통합 AX 인터페이스"**

> - 안전한 사람 허락 시스템

**레고 블록**은 **재조립**이 쉬워요!
