## 목표 및 범위

M1의 목표는 프로젝트의 방향성을 확립하고, 기술적 실현 가능성을 검증하는 것이다. 문제정의서를 통해 산업체 현장의 민원 처리 과정에서 발생하는 병목을 정의하고, PRD를 작성하여 요구사항을 구조화했다.

**범위**:

- 산업체 현장 분석 및 문제 정의
- 시스템 요구사항 도출 및 PRD 작성
- WBS(Work Breakdown Structure) 수립
- 기술 스택 비교 분석 및 최종 선정

## 핵심 산출물

### 문제정의서

폐쇄망 환경에서 민원을 분석하고 처리해야 하는 현장의 요구사항을 정리했다. 클라우드 API 의존 없이 로컬에서 LLM을 실행해야 한다는 핵심 제약 조건을 도출했다.

### PRD (Product Requirements Document)

- 기능 요구사항 15개, 비기능 요구사항 8개 정의
- 사용자 페르소나 3종 (민원 담당 공무원, 관리자, 시스템 운영자)
- 성공 지표: 민원 초안 생성 정확도 ROUGE-L 0.4 이상, 응답 시간 10초 이내

### WBS

6개 마일스톤, 총 14주 일정으로 구성했다. 각 마일스톤에 명확한 산출물과 완료 기준을 정의했다.

### 기술 스택 비교 분석

| 항목 | 후보 A | 후보 B | 최종 선정 |
|------|--------|--------|-----------|
| 베이스 모델 | LLaMA-3-8B | EXAONE-Deep-7.8B | **EXAONE-Deep-7.8B** |
| 파인튜닝 | Full Fine-tuning | QLoRA (4-bit) | **QLoRA** |
| 양자화 | GPTQ | AWQ (W4A16g128) | **AWQ** |
| 서빙 엔진 | TGI | vLLM | **vLLM** |
| 벡터 검색 | ChromaDB | FAISS IndexFlatIP | **FAISS** |
| 임베딩 모델 | KoSimCSE | multilingual-e5-large | **multilingual-e5-large** |

EXAONE-Deep-7.8B를 선정한 이유는 한국어 성능이 우수하고, LG AI Research에서 제공하는 공식 라이선스가 학술/비상업 용도에 적합했기 때문이다.

## 결과 및 회고

### 잘된 점

- 문제정의서를 통해 프로젝트 방향이 명확해졌다
- PRD에 정량적 성공 지표를 포함하여 이후 마일스톤에서 객관적 평가가 가능해졌다
- 기술 스택 비교 분석을 체계적으로 수행하여 이후 기술 변경 리스크를 최소화했다

### 개선할 점

- 초기 일정 산정이 다소 낙관적이었다. M2에서 데이터 전처리 작업량을 과소평가했다
- 산업체 현장 방문 일정 조율에 시간이 소요되어 기획 단계가 예상보다 길어졌다

### 다음 마일스톤 연결

M1에서 수립한 기술 스택과 아키텍처를 바탕으로, M2에서 실제 데이터 파이프라인 구축과 모델 파인튜닝을 진행한다.
